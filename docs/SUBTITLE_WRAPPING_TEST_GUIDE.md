# 字幕自动换行 - 测试指南

## 文档目的

本文档提供字幕自动换行功能的完整测试场景、测试步骤和验收标准，确保功能在各种情况下都能正常工作。

## 测试环境准备

### 1. 浏览器要求

- Chrome 90+ / Edge 90+ / Firefox 88+ / Safari 14+
- 支持 Canvas API 和 `measureText()`

### 2. 测试文件准备

准备以下测试视频和字幕：

- **短视频** (30秒): 用于快速测试
- **长视频** (5分钟): 用于性能测试
- **不同分辨率**: 1920x1080, 1280x720, 3840x2160

### 3. 进入测试页面

1. 访问 `http://localhost:3000/editor-pro`
2. 上传测试视频
3. 创建或导入字幕

## 测试场景清单

### 场景 1: 长句子（中文）

#### 测试目标
验证中文长句能够在合适的标点符号处换行

#### 测试步骤

1. 创建新字幕
2. 输入以下文本:
   ```
   这是一段很长的中文字幕文本，用于测试字幕自动换行功能是否能够正确处理长句子的情况。
   ```
3. 设置字体大小为 32px
4. 设置最大宽度为 80%
5. 观察预览效果

#### 预期结果

- 字幕应在逗号或句号处换行
- 不应超出视频边界
- 标点符号保留在行尾
- 换行后可读性良好

#### 示例输出

```
这是一段很长的中文字幕文本，
用于测试字幕自动换行功能是否能够正确处理长句子的情况。
```

## 已知问题和限制

### 当前版本的限制

1. **Emoji 宽度**: 部分 emoji 在不同系统上宽度测量可能不一致
2. **RTL 语言**: 未测试阿拉伯文、希伯来文等从右到左的语言
3. **竖排文字**: 不支持竖排文字（日文竖排等）
4. **复杂字形**: 某些复杂字形（如韩文组合字）可能有细微偏差

### 未来改进方向

1. 增加对 RTL 语言的支持
2. 优化 emoji 宽度测量
3. 支持更多断句规则（如中文专有名词不断开）
4. 增加用户自定义断句点

## 自动化测试建议

    const text = '这是一段很长的字幕文本';

  it('should not wrap short text', () => {
