# 进阶编辑器时间轴交互测试验证报告

## 概述
本文档提供了进阶编辑器（editor-pro）时间轴字幕交互功能的全面测试场景和验证方法。

## 测试环境准备

### 1. 启动应用
```bash
npm run dev
```

### 2. 进入进阶编辑器
- 访问 `/editor-pro` 页面
- 上传一个测试视频
- 导入或创建至少 2 个轨道的字幕

### 3. 打开浏览器控制台
按 F12 打开开发者工具，切换到 Console 标签页。你应该会看到类似以下的测试场景表格：


## 测试场景详解

### 场景 1: 单击第一轨道字幕 → 应该选中

**操作步骤:**
1. 在时间轴找到第一个轨道的任意字幕块
2. 单击字幕块的中间区域
3. 快速释放鼠标，不要移动

**预期结果:**
- 字幕块高亮显示（边框变亮、发光效果）
- 播放头跳转到字幕开始时间
- 右侧属性面板显示该字幕的详细信息

**控制台日志示例:**
```
🖱️ MouseDown | 移动区
  轨道[中文字幕]
  字幕[abc12345:"这是测试字幕..."]
  位置: (450, 280)
  时间: 1.50s - 3.20s

✅ MouseUp   | 选中字幕
  距离: 2px
  判定: 点击

🎯 Click     | 新选中
  轨道[中文字幕]
  字幕[abc12345:"这是测试字幕..."]
  时间: 1.50s
```

**故障排查:**
- 如果字幕没有被选中，检查是否不小心移动了鼠标
- 如果看到 "判定: 拖拽"，说明移动距离超过了 5px 阈值

---

### 场景 2: 单击第二轨道字幕 → 应该选中

**操作步骤:**
1. 在时间轴找到第二个轨道的任意字幕块
2. 单击字幕块的中间区域
3. 快速释放鼠标，不要移动

**预期结果:**
- 第二轨道的字幕块高亮显示
- 之前选中的字幕取消高亮
- 播放头跳转到该字幕的开始时间
- 当前选中轨道切换到第二轨道

**控制台日志示例:**
```
🖱️ MouseDown | 移动区
  轨道[英文字幕]
  字幕[def67890:"This is a test..."]
  位置: (450, 340)
  时间: 2.10s - 4.50s

✅ MouseUp   | 选中字幕
  距离: 1px
  判定: 点击

🎯 Click     | 新选中
  轨道[英文字幕]
  字幕[def67890:"This is a test..."]
  时间: 2.10s
```

**关键验证点:**
- **这是修复的核心场景** - 之前第二轨道点击不生效的问题已解决
- 第二轨道字幕应该和第一轨道字幕一样，点击即可选中
- 不应该出现任何延迟或无响应的情况

**故障排查:**
- 如果第二轨道字幕无法选中，检查控制台是否有错误
- 确认是否看到了 MouseDown、MouseUp、Click 三个日志
- 如果只看到 MouseDown 但没有 Click，说明被判定为拖拽

---

### 场景 3: 水平拖拽字幕 → 应该调整时间

**操作步骤:**
1. 在时间轴找到任意字幕块
2. 在字幕块中间区域按下鼠标
3. 水平向左或向右拖拽至少 10px
4. 释放鼠标

**预期结果:**
- 字幕块在时间轴上左右移动
- 字幕的开始时间和结束时间同步调整
- 字幕持续时长保持不变

**控制台日志示例:**
```
🖱️ MouseDown | 移动区
  轨道[中文字幕]
  字幕[abc12345:"测试字幕"]
  位置: (500, 280)
  时间: 3.00s - 5.00s

🔄 MouseMove | 拖拽中
  距离: 25px (X:25px, Y:2px)
  方向: 未确定
  类型: move

↔️ Direction | 水平拖拽
  X偏移: 50px
  Y偏移: 3px

🔄 MouseUp   | 调整时间
  距离: 50px
  判定: 拖拽
```

**验证方法:**
- 拖拽前记录字幕的时间范围
- 拖拽后检查时间是否改变
- 验证持续时长 = 结束时间 - 开始时间 保持一致

**故障排查:**
- 如果字幕没有移动，检查是否看到了 "Direction | 水平拖拽"
- 如果方向被判定为垂直，说明 Y 轴移动过大

---

### 场景 4: 垂直拖拽字幕 → 应该切换轨道

**操作步骤:**
1. 确保有至少 2 个轨道
2. 选择第一个轨道的字幕块
3. 在字幕块中间区域按下鼠标
4. 垂直向上或向下拖拽至少 50px
5. 释放鼠标

**预期结果:**
- 字幕从源轨道移除
- 字幕出现在目标轨道
- 字幕的时间保持不变
- 字幕的其他属性（颜色、样式等）保持不变

**控制台日志示例:**
```
🖱️ MouseDown | 移动区
  轨道[中文字幕]
  字幕[abc12345:"测试字幕"]
  位置: (500, 280)
  时间: 3.00s - 5.00s

🔄 MouseMove | 拖拽中
  距离: 52px (X:5px, Y:52px)
  方向: 未确定
  类型: move

↕️ Direction | 垂直拖拽
  X偏移: 5px
  Y偏移: 52px

🔀 TrackSwitch
  从: 轨道[中文字幕]
  到: 轨道[英文字幕]

🔀 MouseUp   | 切换轨道
  距离: 52px
  判定: 拖拽
  目标: 轨道[英文字幕]
```

**验证方法:**
- 拖拽前记录字幕所在的轨道
- 拖拽后检查字幕是否在新轨道
- 确认时间没有改变

**故障排查:**
- 如果字幕没有切换轨道，检查 Y 轴移动是否超过 30px
- 确认 Y 偏移是否大于 X 偏移的 2 倍
- 检查目标轨道是否存在

---

### 场景 5: 快速点击 → 应该选中，不触发拖拽

**操作步骤:**
1. 在时间轴找到任意字幕块
2. 快速点击字幕块中间区域（按下并立即释放）
3. 确保鼠标移动距离小于 5px

**预期结果:**
- 字幕被选中
- 不会触发拖拽操作
- 字幕位置和时间不变

**控制台日志示例:**
```
🖱️ MouseDown | 移动区
  轨道[中文字幕]
  字幕[abc12345:"测试字幕"]
  位置: (500, 280)
  时间: 3.00s - 5.00s

✅ MouseUp   | 选中字幕
  距离: 0px
  判定: 点击

🎯 Click     | 新选中
  轨道[中文字幕]
  字幕[abc12345:"测试字幕"]
  时间: 3.00s
```

**关键验证点:**
- 距离应该是 0px 或 1-2px
- 判定应该是 "点击"
- 应该看到 Click 事件

**故障排查:**
- 如果被判定为拖拽，可能是鼠标晃动导致
- 尝试更快速地释放鼠标
- 使用触控板可能比鼠标更容易控制

## 判定标准详解

### 点击 vs 拖拽
- **移动距离 < 5px** → 判定为点击
- **移动距离 ≥ 5px** → 判定为拖拽

### 水平拖拽 vs 垂直拖拽
只在 `dragType === 'move'` 时判断方向：

- **垂直拖拽条件:**
  - Y 轴偏移 > 30px
  - 且 Y 轴偏移 > 2 × X 轴偏移

- **水平拖拽条件:**
  - 不满足垂直拖拽条件的其他情况

### 边缘拖拽
- **左边缘拖拽** (`dragType === 'left'`) → 始终为水平拖拽，调整开始时间
- **右边缘拖拽** (`dragType === 'right'`) → 始终为水平拖拽，调整结束时间

## 常见问题排查

### Q1: 第二轨道字幕无法选中
**症状:** 点击第二轨道字幕没有反应

**检查步骤:**
1. 打开控制台，查看是否有 MouseDown 日志
2. 检查是否有 Click 日志
3. 查看 `判定` 是否为 "点击"

**可能原因:**
- 鼠标移动超过 5px 被判定为拖拽
- JavaScript 错误阻止了事件处理

**解决方法:**
- 更快速地释放鼠标
- 检查控制台错误日志

---

### Q2: 点击总是被判定为拖拽
**症状:** 控制台显示 "判定: 拖拽" 而不是 "判定: 点击"

**检查步骤:**
1. 查看 MouseUp 日志中的 "距离" 字段
2. 确认距离是否 ≥ 5px

**可能原因:**
- 鼠标稳定性不足
- 触控板灵敏度过高

**解决方法:**
- 使用更稳定的鼠标
- 调整触控板灵敏度
- 按下和释放要更快

---

### Q3: 垂直拖拽不触发轨道切换
**症状:** 向上/向下拖拽字幕，但字幕没有切换轨道

**检查步骤:**
1. 查看是否有 "Direction | 垂直拖拽" 日志
2. 检查 Y 偏移是否 > 30px
3. 检查 Y 偏移是否 > 2 × X 偏移

**可能原因:**
- Y 轴移动不够
- X 轴移动过多

**解决方法:**
- 尽量垂直拖拽，减少水平移动
- 拖拽距离超过 50px

---

### Q4: 水平拖拽时字幕跳动
**症状:** 拖拽字幕时，字幕位置突然跳动

**检查步骤:**
1. 查看 MouseDown 日志
2. 检查是否记录了 clickOffset

**可能原因:**
- clickOffsetTime 计算错误

**解决方法:**
- 从字幕块左侧开始拖拽
- 报告 bug 给开发团队
