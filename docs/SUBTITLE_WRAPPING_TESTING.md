# 字幕自動換行測試指南

## 快速測試清單

### 1. 基本換行測試 (5分鐘)

#### 測試用例 1：長中文句子
```
有時候，我覺得我需要一個女孩，然後我得到了一個，
```
**預期結果**：
- ✅ 應該在逗號處自動換行
- ✅ 不超出螢幕邊界
- ✅ 類似這樣：
  ```
  有時候，我覺得我需要一個女孩，
  然後我得到了一個，
  ```

#### 測試用例 2：中英文混合
```
This is a very long subtitle text with both English and 中文字符混合在一起的內容
```
**預期結果**：
- ✅ 根據實際寬度自動換行
- ✅ 優先在空格或標點符號處斷開

#### 測試用例 3：無標點的長句
```
這是一段沒有任何標點符號的超級長句子用來測試強制斷行的情況看看會不會正確處理
```
**預期結果**：
- ✅ 強制在接近最大寬度處斷開
- ✅ 不超出容器

### 2. 字體樣式測試 (3分鐘)

1. **修改字體大小**
   - 設置 24px → 觀察換行
   - 設置 48px → 觀察換行
   - 設置 72px → 觀察換行
   - **預期**：字體越大，每行容納的字符越少

2. **修改字體族**
   - Arial (默認)
   - Microsoft YaHei (微軟雅黑)
   - 宋體
   - **預期**：不同字體寬度不同，換行位置應該調整

3. **修改字重**
   - Normal → Bold
   - **預期**：粗體更寬，每行字符應該減少

### 3. 響應式測試 (2分鐘)

1. **調整窗口寬度**
   - 縮小瀏覽器窗口 → 觀察字幕換行增多
   - 放大瀏覽器窗口 → 觀察字幕換行減少
   - **預期**：動態適應窗口寬度

2. **調整字幕容器寬度**
   - 在屬性面板修改 "最大寬度" (maxWidth)
   - 從 80% → 60% → 40%
   - **預期**：容器越窄，換行越多

### 4. 固定字幕測試 (2分鐘)

1. **頂部標題**
   - 添加固定字幕（頂部）
   - 輸入長標題：`【震撼】這個女孩的故事讓所有人都沉默了，你絕對想不到結局是什麼`
   - **預期**：自動換行，不超出螢幕

2. **底部浮水印**
   - 添加固定字幕（底部）
   - 輸入長文本測試

### 5. 導出測試 (5分鐘)

1. **導出視頻**
   - 點擊 "錄製預覽" 或 "WebCodecs 導出"
   - 等待完成

2. **播放檢查**
   - 下載並播放導出的視頻
   - **檢查項**：
     - ✅ 字幕換行與預覽一致
     - ✅ 不超出視頻邊界
     - ✅ 斷句位置合理

## 常見問題排查

### 問題 1：字幕仍然超出螢幕

**可能原因**：
1. 字體未載入完成
2. `maxWidth` 設置過大（>100%）
3. 字幕包含超長的單個單詞（無空格）

**解決方法**：
1. 等待頁面完全載入後再測試
2. 檢查 maxWidth 設置（推薦 80%）
3. 手動添加空格或標點符號

### 問題 2：換行位置不理想

**可能原因**：
- 標點符號位置不佳
- 需要手動調整

**解決方法**：
1. 手動在合適的位置插入換行符 (`\n`)
2. 調整標點符號位置

### 問題 3：導出視頻換行與預覽不一致

**可能原因**：
- 視頻分辨率與預覽不同
- 字體在不同環境下的渲染差異

**解決方法**：
1. 檢查導出設置中的分辨率
2. 確認使用系統字體（如 Arial）而非自定義字體

## 性能檢查

### 正常情況
- 字幕切換：< 16ms (60fps)
- 窗口調整：即時響應
- 導出速度：與原來相同

### 異常情況（需要優化）
- 字幕切換卡頓
- 窗口調整延遲 > 100ms

**聯繫開發者**並提供：
1. 字幕數量
2. 字幕平均長度
3. 瀏覽器版本
4. 操作系統
